The Game - Game

C++ - SDL Libraries

A beautiful game following the life of a bouncy ball on his path to find true love. (You might be thinking: "Bouncy balls can't love, they are bouncy balls." While probably true, I was unable to find a study about the capacity of bouncy balls to love, and so I cannot say anything for certain.)

The majority of the code lies in creating the interface with the renderer. The important things that had to be tracked were the geometry of the view window with respective to the level's video memory, the geometry of each object, as well as detecting collisions between objects. Once the level was created, I needed to consider the physics of the ball which consisted of its own velocity, acceleration, and reactions to different collisions.

Finally any stats that want to be displayed can be written to the screen, such as the number of deaths, or text can also be used to tell a beautiful story.

UpNext Group Jukebox

HTML - AngularJS - NodeJS - Mopidy

A collaborative playlist queue with spotify integration created at the corporate Viasat 2015 Hackathon.

The jukebox works based on a system of voting for tracks within a shared playlist. This is geared at music choice within group settings. The order of the next played song in the queue is based off of the users' votes on songs.

The app consisted of a front-end built with HTML and AngularJS and a back-end node.JS server that handled interactions with an external Mopidy server.

My contribution to the project was the back-end Node.JS server. The server handled and processed requests to interact with the music server as well as manage the song queue.

Music Swap

Java - NodeJS - mongoDB

A artist-sharing (Music) android app based around matching users with similar preferences. Users that had similar musical tastes are able to recommend artists to one another. The reason behind this was that the reasons that we love artists aren't always due to the musical style but rather meaning behind songs. In this case, users can be the best way to find new music.

The front-end was an Android app that allowed the user to set up a profile as well as send requests to the server to find matches based on the profile. The front-end used the LastFM API to request and store album and artist art to enhance the visuals of the app. A websocket wrapper for Java was also used to communicate with the Node.JS back-end.

The Node.JS server's main purpose was to manage the mongoDB server. The node.JS server would find two clients who had matching artists and send a reply back to the clients that they were connected. Following a connection clients were able to send artist names to eachother to share music tastes.
